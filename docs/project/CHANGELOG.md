## [Unreleased]

### Fixed
- **pgBouncer auth_type `scram-sha-256` → `md5`**: Los hashes SCRAM en `userlist.txt` tenían salts diferentes a los de PostgreSQL, causando `FATAL: password authentication failed for user "antigravity"` en cada conexión vía pgBouncer. Cambio a `md5` permite que pgBouncer use el password plaintext del database config para autenticarse contra PostgreSQL (generando SCRAM-SHA-256 fresco que sí coincide). Regenerados hashes MD5 para `antigravity` y `app_user` (`md5sum password123+username`).
- **monitoring-service: `lib/pq` → `pgx`**: `lib/pq` v1.10.9 no soporta SCRAM-SHA-256 y está en maintenance mode (sus mantenedores recomiendan pgx). Reemplazado por `jackc/pgx/v5/stdlib` (`_ "github.com/jackc/pgx/v5/stdlib"`, driver name `"pgx"`). Cambio de 2 líneas en `main.go`.
- **Migración faltante `provision_logs` + `custom_charts`**: Estas tablas eran creadas por Alembic (`stats-service/migrations/001_provision_logs.py`) que NUNCA se ejecutaba en K8s. `Base.metadata.create_all()` fue eliminado correctamente de stats/typing, pero la migración equivalente en node-pg-migrate nunca se escribió (finding C3 marcado como resuelto incorrectamente). Nueva migración `17738000000115_create_stats_tables.cjs` con `CREATE TABLE IF NOT EXISTS` y tipos modernos (`GENERATED BY DEFAULT AS IDENTITY`, `TIMESTAMPTZ`, `JSONB`). Migration 13 (`fase2_modelo`) actualizada para excluir estas tablas de sus ALTERs.
- **Dev overlay `DATABASE_URL` faltantes**: `stats-service-secrets` solo tenía `JWT_SECRET`, `typing-service-secrets` no tenía parche. Ambos apuntaban a `app_user@pgbouncer:6432` (base) sin override en dev. Agregados `DATABASE_URL` directos a `postgres:5432` en ambos parches. `monitoring-service-secrets` también recibió `DATABASE_URL` con `sslmode=disable`.
- **`POSTGRES_DB: antigravity` → `vcenter_provisioner`**: El overlay dev creaba una base de datos fantasma `antigravity` (7.5MB, vacía) como artefacto del `POSTGRES_DB` incorrecto. Corregido al valor real.

### Changed
- **migration 13 (`fase2_modelo`)**: Excluye `provision_logs` y `custom_charts` de las listas de ALTER (SERIAL→IDENTITY, TIMESTAMP→TIMESTAMPTZ, JSON→JSONB) porque la nueva migración 11.5 ya las crea con tipos correctos.

## [0.3.0] - 2026-03-19

### Renombramiento de Servicios

| Servicio Anterior | Servicio Nuevo |
|------------------|---------------|
| `vcenter-config-service` | `credential-manager` |
| `vcenter-integration` | `vcenter-operations` |
| `provisioner-vcenter-config` | `provisioner-credential-manager` |
| `provisioner-vcenter-integration` | `provisioner-vcenter-operations` |
| `antigravity/vcenter-config` | `antigravity/credential-manager` |
| `antigravity/vcenter-integration` | `antigravity/vcenter-operations` |

### Cambio de Puertos
- credential-manager: 8082 → **8090**
- vcenter-operations: 8081 → **8091**

### Variables Deprecadas
- `VCENTER_CONFIG_URL` → `CREDENTIAL_MANAGER_URL`
- `VCENTER_INTEGRATION_URL` → `VCENTER_OPERATIONS_URL`

### Fixed
- Pruebas de conexión vCenter corregidas: flujo correcto con token de sesión vía `/api/session`.
- Modo "insecure" (`allowInsecure`) para omitir validación TLS en pruebas de conexión.
- Correcciones TypeScript en credential-manager (imports ESM, try/catch, softDelete).
- Lint: `tsc: not found` en auth-service por node_modules incompletos.

### Added
- **Sistema de migraciones** con node-pg-migrate: 6 migraciones idempotentes (users, vcenter_connections, typification, vm_classes, vm_provisions, audit_logs). Pipeline: `./pipeline.sh --migrate`.
- **API Gateway**: proxy público para `/monitoring`.
- **Frontend**: null/undefined guards en useServiceMonitor, useMonitoringHistory, ServiceDiagram, MonitorPage.

## [0.2.0] - 2026-02-06

### Documentación Consolidada
- `ARCHITECTURE.md` unifica 4 documentos de arquitectura. `CI-CD-LOCAL.md` unifica documentación CI/CD.
- Actualizados: `db-schema.md`, `ux-specification.md`, `review-briefing.md`, `dos-and-donts-playbook.md`.

### Added — Sistema de Monitoreo
- **Shared Scripts**: imagen `antigravity/shared-scripts` con `probe-scheduler.sh` (modo full/sample) y wrappers para Go/Python/Node/Nginx.
- **9 Dockerfiles** actualizados con multi-stage COPY desde shared-scripts.
- **Monitoring Service**: endpoint `POST /api/probe-result`, almacenamiento en Redis + PostgreSQL.
- **Provisioner UI**: nueva página `/monitor` con ServiceDiagram, ServiceCard, useServiceMonitor.
- **Variables PROBE_\*** en todos los servicios (intervalo, modo, sample count, targets).

### Fixed
- nginx-unprivileged: wrapper simple sin bash. Probe scheduler detecta HTTP 200 sin body JSON.
- Dockerfiles: corregidos COPY de archivos y creación de usuario clouduser.
- Variable `SCRIPTS_HASH` → `SHARED_SCRIPTS_HASH` en docker-compose.yml.
